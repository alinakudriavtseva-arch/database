### 1. Опис предметної області та ER-модель
Система «Автосервіс» призначена для обліку ремонтних робіт.
Основні сутності:
* `Замовлення` — зв'язує клієнта, авто та роботи.
* `Виконавці` — проміжна таблиця для зв'язку майстрів із замовленнями.
* `Замовлені_Послуги` та `Використані_Деталі` — деталізація замовлення.

### 2. Запити мовою реляційної алгебри

#### Запит 1: Перелік робіт майстра за період
**Завдання:** Знайти всі послуги, які виконав майстер (ID_M) за певний період, та їх вартість.

**Алгоритм виконання:**
1.  Фільтруємо замовлення за часом:
    `T1 := Замовлення WHERE Дата >= D_Start AND Дата <= D_End`
2.  Знаходимо замовлення конкретного майстра:
    `T2 := Виконавці WHERE ID_Майстра = ID_M`
3.  Об'єднуємо замовлення майстра з часовим періодом:
    `T3 := T1 JOIN T2`
4.  Додаємо інформацію про послуги в цих замовленнях:
    `T4 := T3 JOIN Замовлені_Послуги`
5.  Додаємо назви послуг з довідника:
    `T5 := T4 JOIN Послуга`
6.  Результат (назва, кількість, ціна):
    `T6 := T5 [Назва_послуги, Кількість, Вартість]`

---

#### Запит 2: Витрати деталей майстром
**Завдання:** Скласти список деталей, використаних майстром (ID_M) у замовленнях за період.

**Алгоритм виконання:**
1.  Фільтруємо замовлення за датою:
    `T1 := Замовлення WHERE Дата >= D_Start AND Дата <= D_End`
2.  Фільтруємо таблицю виконавців за майстром:
    `T2 := Виконавці WHERE ID_Майстра = ID_M`
3.  Отримуємо ID замовлень цього майстра за цей час:
    `T3 := T1 JOIN T2`
4.  З'єднуємо з таблицею використаних деталей:
    `T4 := T3 JOIN Використані_Деталі`
5.  Отримуємо назви деталей з довідника:
    `T5 := T4 JOIN Деталь`
6.  Результат:
    `T6 := T5 [Назва_деталі, Кількість, Вартість_деталі]`
